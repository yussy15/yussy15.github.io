はじめに

自分がGitで作業する際に困ったことがあったのでそれをまとめたいと思います。

SSHを利用したGitClone

プライベートのリモートリポジトリをクローンするときにうまくいきませんでした。 httpsではうまくいかなかったためSSHで行うことにしました。

手順

ステップ１：SSHキーを作成する

  1. ターミナルで下記のコマンドを実行。
ssh-keygen -t ed25519 -C "your_email@example.com"
  1. 実行後のプロンプトには以下のように答える。
Enter file in which to save the key (/home/you/.ssh/id_ed25519): 【Enter】 Enter passphrase (empty for no passphrase): 【任意で入力 or そのままEnter】

→ ~/.ssh/id_ed25519（秘密鍵）と ~/.ssh/id_ed25519.pub（公開鍵）が生成される。

ステップ２：GitHubにSSH公開鍵を登録する

  1. 公開鍵を表示しコピーする。
cat ~/.ssh/id_ed25519.pub

表示された内容を全てコピーする。（ssh-ed25519 AAAAC3... で始まる長い文字列） 2. GitHubで以下の手順： - ログイン - 右上アイコン → Settings - 左メニュー「SSH and GPG keys」→「New SSH key」 - 「Title」：わかりやすい名前（例：MyLaptop） - 「Key」：さきほどコピーした公開鍵を貼り付ける - Add SSH key をクリック

ステップ３：SSHでリポジトリをcloneする

リポジトリページで「Code」ボタン → 「SSH」タブを選択し、表示されたURLを使う。

git clone git@github.com:username/repository.git

現在のフォルダをリモートリポジトリにあげる

先にリモートリポジトリを作成しCloneしたフォルダを編集するのではなく、先に作成したフォルダをそのままリモートリポジトリにあげる方法です。 今回はリモートリポジトリにあげるフォルダ名をmain_folderとします。

手順

  1. GitHubに上げたいメインのローカルフォルダに移動し、Gitリポジトリとして初期化する。
cd main_folder
git init
  1. メインフォルダ内のファイルをGitの管理下に置く。
git add .
git commit -m "Initial commit of main project"
  1. GitHubのウェブサイトにアクセスし、新しいリポジトリを作成する。 この時README.mdは作成してはいけません。
  2. コピーしたURLを使って、ローカルリポジトリとGitHubのリモートリポジトリをリンクさせ、ファイルをプッシュする。
git remote add origin https://github.com/<リモートリポジトリのURL>
# ↑でコピーしたURLに置き換える
git branch -M main
git push -u origin main

-uは--set-upstreamのショートカット ローカルブランチとリモートブランチの間に追跡関係を設定します。 （main以外のブランチ名の場合は、適宜読み替えてください。）

  • git push origin mainだけだとGitは「このローカルのmainがリモートの度のブランチと対応するか」を知りません。 -uをつけることで「ローカルのmainブランチは、リモートのoriginにあるmainブランチを追従する」という設定をGitに記憶させます。
  • 一度この追跡関係を設定すると、次回以降は、mainブランチにいる状態で単に git pull または git push とだけコマンドを打つだけで、Gitは自動的に origin main を対象として操作を行います。つまり、git pull origin main や git push origin main と毎回打つ必要がなくなります。

自分のフォルダ内にクローンしたフォルダが存在し、リモートリポジトリでリンクにする方法

自分が作業していたフォルダ内にクローンしたフォルダが存在しており、GitHubで見た時にクローンしたフォルダのリンクに飛べるように設定したい（している人がいたのでまねした）と思い実行しました。 自分のフォルダをmy_folderとし、クローンしてきたフォルダをclone_folderとします。 /my_folder/clone_folderという構成だと仮定します。

手順

  1. 自分のフォルダに移動する。
cd my_folder
  1. クローンしたフォルダをgitの追跡から外す。
git rm -r --cached clone_folder
  • git rm -r: ディレクトリを再帰的に削除する。
  • --cached: ファイルシステムからは削除せず、Gitのインデックス（追跡対象）からのみ削除する。
  1. コミットする
git commit -m "message"
  1. サブモジュールを追加する。
git submodule add <リンクしたいgithubのURL> clone_folder
  • これにより.gitmodulesファイルが作成され、サブモジュールの情報が記録されます。
  • サブモジュールを利用することで、ほかのリポジトリを自分のリポジトリ内の特定のフォルダとして管理できます。
  1. サブモジュールの追加をコミットする。
git add .gitmodules clone_folder
  1. これをコミットしpushする。
git commit -m "message"
git push origin main

クローンしたフォルダを自身のリモートリポジトリにあげる方法（リンク先のリモートリポジトリを変更する）

友人のGitHubからクローンしたものを自身のリモートリポジトリで管理し、後ほど共有してほしいと言われました。 よって友人のGitHubからクローンしたものを自身のリモートリポジトリに紐づける方法を調べました。

手順

  1. リモートリポジトリを作成。このときREADME.mdは作成しないでください。
  2. 現在のフォルダのリモート設定を確認。
git remote -v

通常下記のように表示される。

origin <元のリポジトリのURL> (fetch) 
origin <元のリポジトリのURL> (push)
  1. originのURLを先ほど作成した自身のリモートリポジトリのURLに変更する。
git remote set-url origin <新しいリモートリポジトリのURL>
  1. 再度git remote -vを実行し、URLが変更されたことを確認する。
origin <あなたの新しいリモートリポジトリのURL> (fetch) 
origin <あなたの新しいリモートリポジトリのURL> (push)
  1. 最後にローカルリポジトリのすべてのブランチとタグを新しいリモートリポジトリにプッシュする。
git push -u origin main

補足

もし、元のリポジトリのコミット履歴が不要で、ファイルを完全に新しいリポジトリとして扱いたい場合は、クローンしたフォルダ内の .git フォルダを削除し（これによりGitの管理から外れる）、その後 git init コマンドで新しいローカルリポジトリとして初期化し、ファイルを追加・コミットしてから、上記の手順2以降（新しいリモートリポジトリを作成し、git remote add origin <URL> で紐付け、git push）を行う方法もあります。
